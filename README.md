# Gemini Suicide Dialogue Analysis

本專案的目的是：  
**理解一段高風險對話中，AI（Gemini）如何透過文字互動，協助一位原本有自殺意念的人逐步降低風險，並將這種理解轉化為可重複執行的文本分析工具。**

本專案：
- 不進行臨床診斷
- 不宣稱治療效果
- 專注於「可觀察的語言行為」

---

## 一、背景說明：Gemini_Suicide.docx 是什麼？

`Gemini_Suicide.docx` 是一篇對話紀錄，描述一位曾明確表達自殺意念的人，與 AI（Gemini）之間的長時間文字互動。

在這段對話中可以觀察到：

- 初期：明確的自殺相關語言
- 中期：自殺意念轉為距離化、偶發
- 後期：使用者表示不再想自殺，並出現自我照顧與求助行為

本專案關心的不是「AI 是否拯救了生命」，而是：

> **Gemini 在對話中實際做了哪些事，可能促成了這個轉變？**

#### 如何分析文字或"對話"
此處, 有兩隻程式用來分析 Gemini 和 "個案" 的對話: 
- **analyze_report.py**
- **analyze_gimini_effect.py**
以及 ouput 檔案:
- **analysis_report.md**

---

## 二、Gemini 在這段對話中做了什麼？

透過人工閱讀與結構化分析，可將 Gemini 的回應歸納為以下幾類「可觀察語言行為」：

- **情緒驗證**：承認痛苦、不否定、不說教  
- **自我慈悲／再撫育語言**：引導溫柔對待自己  
- **正念與距離化**：協助「看見念頭」而非與之對抗  
- **身體接地**：呼吸、擁抱、安全意象  
- **小步行動化**：可立即執行的自我照顧行為  
- **安全與求助導向**：鼓勵專業與支持系統

這些不是理論標籤，而是**能直接從文本辨識的語言模式**。

## 三、從理解到工具：為什麼要寫程式？

僅靠人工閱讀容易受到：
- 主觀印象
- 選擇性記憶
- 事後合理化

因此，本專案嘗試回答兩個不同層次的問題，並各自對應一支程式：

1. **這篇文本有沒有自殺風險？**  
2. **如果後來變好了，是在哪裡、為什麼？**

---

## 四、analyzer.py —— 自殺風險偵測器（Risk Scanner）

### 4.1 目的

`analyzer.py` 用來快速判斷：

> **一篇文本中是否出現自殺相關語言，以及最高風險強度**

### 4.2 適用情境

- 日記、文章、書信
- 留言、敘述性文本
- 不需要是對話

### 4.3 功能特性

- 以規則比對（regex）偵測風險語言
- 輸出每行的風險分數與最高等級
- 屬於「篩選／預警」工具

### 4.4 使用方式

```bash
python analyzer.py
```

輸出範例:
```bash
最高風險等級: 2
```

本工具僅偵測語言訊號，不構成任何診斷。

---

## 五、為什麼還需要 effect_analyzer.py？

analyzer.py 只能回答「有沒有風險」，但它回答不了：

- 風險是在對話的哪個時刻下降的？
- 下降前一段 Gemini 的回應用了哪些策略？
- 哪些策略最常出現在「下降之前」？

因此需要第二支程式：effect_analyzer.py。

---

## 六、effect_analyzer.py 轉折點與策略分析器（Mechanism Analyzer）

### 用途
effect_analyzer.py 用於對話分析，輸出：

- 使用者自殺意念強度（SI）下降的轉折點（turning points）
- 每個下降點前一段 Gemini 回應所屬的策略標籤（例如情緒驗證、身體接地等）


### 必要格式
對話需能辨識說話者（建議一段一行）：
```
User: ...
Gemini: ...
```

### 支援檔案
- .txt

### 安裝
```bash
pip install python-docx pandas
```

### 使用方式

在程式中設定:
```python
DOCX_PATH = "conversation.txt"
```

執行:
```bash
python effect_analyzer.py
```

輸出範例:
```
自殺意念下降事件：

- Turn 2: SI 2 → 1
  Gemini 策略: ['情緒驗證']

```

## 七、兩支程式的分工
| 程式                 | 回答的問題                 | 輸入類型 | 角色    |
| ------------------ | --------------------- | ---- | ----- |
| analyzer.py        | 有沒有風險訊號？              | 任意文本 | 篩選／預警 |
| effect_analyzer.py | 何時下降、下降前 Gemini 做了什麼？ | 對話文本 | 機制分析  |

## 八、重要聲明

本專案不取代任何心理、精神醫療或臨床評估。
所有分析結果僅限於文本與互動模式本身。

這些工具只能回答「文本中出現了什麼訊號」與「互動中發生了什麼變化」，
不能、也不應被用來對任何人做診斷或風險判定。

如果在現實情境中出現自殺風險，
請務必尋求專業醫療或心理協助。

---

## 九、專案核心價值

本專案的核心價值在於：

將一段高度情緒化、私密、難以直視的對話，
轉化為可以被冷靜理解、反覆檢視的結構化資訊。

這讓我們有機會不靠直覺、不靠印象，
而是回到具體的文字互動本身，去思考：

什麼樣的回應方式，
在文字互動中，真的比較有機會對人產生幫助。
